WSL2 + Docker Desktop 설치 가이드 (Windows 11, 클릭 단위 상세 안내)
=====================================================================

이 문서는 AI 쇼핑 어시스턴트 프로젝트의 Phase 15(배포) 작업을 위해
Docker Desktop(WSL2 백엔드, GPU 패스스루 포함)을 처음 설치하는 사용자를
위한 안내문입니다.

시작 전에 확인할 것
---------------------
- Windows 버전 확인: Win키 + R -> winver 입력 -> Enter. Windows 11이면 OK.
- 여유 디스크 공간: 최소 20GB 이상 권장 (WSL2 + Docker + CUDA 베이스
  이미지 + 프로젝트 모델 캐시 포함).
- 전체 과정은 재부팅이 2번 정도 필요하고, 다운로드/설치 시간까지 합치면
  30분~1시간 정도 걸릴 수 있습니다.


1단계 - WSL2 설치
---------------------
1. 시작 버튼을 클릭하거나 Win키를 누릅니다.
2. "powershell" 이라고 타이핑합니다. 검색 결과에 "Windows PowerShell"
   앱이 뜹니다.
3. 그 앱 위에서 마우스 오른쪽 버튼 클릭 -> 메뉴에서
   "관리자 권한으로 실행" 클릭.
4. "이 앱이 디바이스를 변경할 수 있도록 허용하시겠습니까?" UAC 창이
   뜨면 "예" 클릭.
5. PowerShell 창이 열립니다. 아래를 정확히 입력하고 Enter:

       wsl --install

6. 자동으로 여러 줄의 설치 로그가 지나갑니다(WSL 커널 업데이트, Ubuntu
   배포판 다운로드 등). 인터넷 속도에 따라 몇 분 걸릴 수 있습니다.
7. 설치가 끝나면 "다시 시작해야 합니다" 같은 문구가 나옵니다. PowerShell
   창을 닫고 Windows를 재부팅하세요. (시작 메뉴 -> 전원 버튼 -> 다시 시작)

재부팅 후
~~~~~~~~~
8. 재부팅되면 자동으로 검은 Ubuntu 터미널 창이 하나 뜰 수 있습니다
   (안 떠도 정상이니 그냥 넘어가도 됩니다). 만약 뜨고
   "Enter new UNIX username:" 이라고 물어보면:
     - 아무 영문 소문자 이름이나 입력 (예: ssafy) 후 Enter
     - 비밀번호를 입력하라고 하면 아무 비밀번호나 입력 (입력해도 화면에
       안 보이는 게 정상, 그냥 타이핑 후 Enter) -> 확인을 위해 한 번 더 입력
     - 이 계정은 Docker 사용 자체에는 필요 없고, Ubuntu 터미널을 직접 쓸
       때만 필요한 것이니 잊어버려도 크게 상관없습니다.
9. 확인 절차: PowerShell을 다시 열고 (이번엔 관리자 권한 없어도 됩니다)
   아래 입력:

       wsl --status

   결과에 "기본 버전: 2" 라고 나오면 성공입니다.


2단계 - Docker Desktop 다운로드 및 설치
------------------------------------------
1. 웹 브라우저(크롬/엣지 등)를 열고 주소창에 다음 주소를 입력해 접속:

       https://www.docker.com/products/docker-desktop/

2. 페이지에 파란색 "Download for Windows" 버튼이 보입니다. (혹시 여러
   옵션이 뜨면 반드시 "Windows - AMD64"를 선택 - Apple Silicon/Mac용이
   아닌 Windows용인지 확인)
3. 다운로드가 끝나면 브라우저 하단(또는 다운로드 폴더)에
   "Docker Desktop Installer.exe" 파일이 생깁니다. 이 파일을
   더블클릭해서 실행합니다.
4. "이 앱이 디바이스를 변경하도록 허용하시겠습니까?" UAC 창이 또 뜨면
   "예" 클릭.
5. "Configuration"이라는 설치 옵션 화면이 뜹니다:
     - "Use WSL 2 instead of Hyper-V (recommended)" 체크박스 - 기본적으로
       체크되어 있을 겁니다. 그대로 체크된 상태로 두세요.
     - "Add shortcut to desktop" 체크박스 - 편한 대로 (체크해도 무방)
     - 오른쪽 아래 "OK" 버튼 클릭.
6. 설치 진행 바가 채워지는 걸 몇 분간 기다립니다. (파일 압축 해제, WSL
   관련 컴포넌트 설치 등)
7. 설치가 끝나면 "Installation succeeded" 화면이 뜨고
   "Close and log out" 버튼이 보입니다. 이 버튼을 클릭하면 Windows에서
   로그아웃되거나 재부팅됩니다. (저장 안 한 작업 있으면 먼저 저장하세요.)


3단계 - Docker Desktop 최초 실행
------------------------------------
1. 재부팅/재로그인 후, 바탕화면의 고래 모양 "Docker Desktop" 아이콘을
   더블클릭하거나, 시작 메뉴에서 "Docker Desktop" 검색 후 실행합니다.
2. "서비스 이용 약관(Subscription Service Agreement)" 화면이 뜹니다.
   하단의 "I accept the terms" 체크박스를 체크하고 "Accept" 버튼 클릭.
3. 로그인 화면이 뜨는데, 로그인은 필요 없습니다. 화면 어딘가(보통 우측
   상단이나 하단)에 있는 "Skip" 또는 "Continue without signing in"
   링크를 클릭해서 넘어가세요.
4. Docker 엔진이 백그라운드에서 켜지는 동안 창 좌측 하단에 상태 표시등이
   있습니다: 빨간 점(꺼짐) -> 노란 점(시작 중) -> 초록 점
   ("Engine running"). 초록색이 될 때까지 1~2분 기다립니다.
5. (선택 확인) 작업표시줄 우측 하단, 시계 옆의 숨겨진 아이콘 화살표(^)를
   클릭하면 고래 모양 Docker 아이콘이 보입니다. 그게 계속 떠 있으면
   Docker가 백그라운드에서 실행 중이라는 뜻입니다.


4단계 - WSL2 통합 설정 확인
-------------------------------
1. Docker Desktop 창 우측 상단의 톱니바퀴(설정) 아이콘 클릭 -> Settings
   화면 진입.
2. 왼쪽 메뉴에서 "General" 클릭 -> "Use the WSL 2 based engine" 체크박스가
   체크돼 있는지 확인 (Windows 11에서는 기본으로 체크되어 있고 회색으로
   비활성화돼 있을 수도 있는데, 그건 정상입니다).
3. 왼쪽 메뉴에서 "Resources" -> "WSL Integration" 클릭.
4. "Enable integration with my default WSL distro" 토글이 켜져 있는지
   확인, 그 아래 목록에 뜨는 배포판(Ubuntu 등)의 토글도 켜져 있는지 확인.
5. 화면 우측 아래(또는 하단)에 "Apply & Restart" 버튼이 보이면 클릭합니다.
   (변경사항이 없으면 버튼이 없을 수도 있음)


5단계 - 정상 설치 확인
--------------------------
1. PowerShell을 새로 엽니다 (관리자 권한 필요 없음, 그냥 열면 됩니다).
2. 아래 입력:

       docker --version

   -> "Docker version 2x.x.x, build ..." 같은 결과가 나오면 성공.
3. 아래 입력:

       docker run hello-world

   -> 처음엔 작은 이미지를 자동으로 다운로드하고, 화면에
   "Hello from Docker!"로 시작하는 메시지가 나오면 Docker가 완전히
   정상 동작한다는 뜻입니다.


6단계 - GPU(그래픽카드) 인식 확인 (이 프로젝트에 꼭 필요)
-----------------------------------------------------------
1. 먼저 Windows에서 그래픽카드가 잘 잡히는지 확인:

       nvidia-smi

   -> RTX 4070 정보가 표 형태로 나오면 호스트 드라이버는 문제없습니다.

2. Docker 컨테이너 안에서도 GPU가 보이는지 확인 - 아래 명령을 PowerShell에
   입력:

       docker run --rm --gpus all nvidia/cuda:12.8.0-base-ubuntu22.04 nvidia-smi

   - 이 명령은 NVIDIA CUDA 이미지를 처음 한 번 다운로드하고 (몇백 MB,
     몇 분 소요), 그 컨테이너 안에서 nvidia-smi를 실행합니다.
   - 컨테이너 안에서도 RTX 4070 정보가 똑같이 출력되면 GPU 패스스루까지
     완벽하게 성공한 것입니다. 이게 되어야 이 프로젝트(YOLOv8/CLIP/
     Llama-3B/KO-SRoBERTa)를 컨테이너 안에서 GPU로 돌릴 수 있습니다.
   - 만약 "could not select device driver" 같은 오류가 뜨면, Docker
     Desktop을 완전히 종료 후 재시작하거나 재부팅을 한 번 더 해보세요.
